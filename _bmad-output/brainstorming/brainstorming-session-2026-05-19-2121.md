# Brainstorming Session Results

**Facilitator:** Kevin
**Date:** 2026-05-19

## Session Overview

**Topic:** Web app for SPOQ partner onboarding - digitizing the full 6-phase onboarding process to replace Excel-based workflows
**Goals:** Replace three Excel workbooks with a centralized web application; automate tracking, checklists, progress and KPI monitoring; make the partner model scalable

### Context Guidance

_Three Excel workbooks currently manage the entire partner lifecycle: (1) Template Partners for partner data, products and pricing, (2) Partner Overview & Onboarding for 6-phase progress tracking across 39 partners, and (3) Partner Success Workbook for KPIs, scorecards, contact logs and customer satisfaction. Key bottlenecks are in Marketing/Communication (Phase 4) and Evaluation (Phase 6). The system needs to scale beyond spreadsheet limitations._

### Session Setup

_Kevin wants to brainstorm ideas for a web application that replaces the current Excel-based partner onboarding and success management workflow. Focus areas include automation, tooling, UX for the BDM/partner workflow, and scalability._

## Technique Selection

**Approach:** AI-Recommended Techniques
**Analysis Context:** Web app for SPOQ partner onboarding with focus on replacing Excel workflows, automation, and scalability

**Recommended Techniques:**

- **First Principles Thinking:** Strip away Excel assumptions to identify fundamental partner onboarding needs before designing the web app
- **Role Playing:** Walk through the app from every stakeholder perspective (BDM, Partner, IT, Marketing, Sales) to surface features and UX needs per role
- **SCAMPER Method:** Systematically apply 7 transformation lenses to the existing 6-phase workflow to innovate rather than merely digitize

**AI Rationale:** The existing Excel workflows encode assumptions that should be challenged before digitization. A multi-stakeholder product requires perspective diversity. The structured SCAMPER finish ensures concrete, actionable transformation decisions.

## Technique Execution Results

**First Principles Thinking:**

- **Interactive Focus:** Stripping away Excel assumptions to identify fundamental truths about partner onboarding
- **Key Breakthroughs:**
  - F1: Automatic detection as foundation, human action as choice -- the system chases, the human decides
  - F2: The 6-phase model is proven and remains the backbone
  - F3: Scope = onboarding tool, not lifecycle platform (stops at go-live)
  - F4: Configurable path based on integration choice (Forms ERP vs Partner Portal)
  - F5: Multi-user system with role-based tasks (BDM, IT, Marketing, Sales, Partner)

**Role Playing:**

- **Building on Previous:** Multi-user insight (F5) explored through each stakeholder's lens
- **New Insights:**
  - BDM1: Alert-driven dashboard -- attention where it hurts
  - BDM2: Blockage-first partner view with context on demand
  - BDM3: External source integrations (Circleback transcripts, DocuSign)
  - IT1: Partner-centered tasks with push notifications on activation
  - Marketing1: Tasks with prerequisite assets -- no more chasing materials
  - Partner1: Own login from contract signing, self-upload assets, track status
  - Partner2: Transparent status view with scheduled dates
  - System1: Task dependencies and automatic handoff between roles
  - System2: Trust-based checkbox completion -- lightweight and fast
  - System3: Deliverable chain -- tasks produce assets other tasks need
  - System4: Partner uploads feed the system and unblock downstream tasks

**SCAMPER Method:**

- **Building on Previous:** Took the 6-phase model and existing tasks through transformation lenses
- **Developed Ideas:**
  - S1: Standard contracts as system choice, lawyer only on exceptions (Expert-Partner / Software-Partner)
  - S2: Contract type as second configuration variable alongside integration choice
  - C1: Combine Phase 4 (Marketing) + Phase 5 (Sales Enablement) into single "Go-to-Market" phase
  - E1: Main tasks with assignable subtasks -- dashboard stays clean, granularity where needed
  - M1: Deadline-driven onboarding with backward planning from target go-live date
  - R1: Self-service partner intake flow replacing contact form as front door to the system

**Overall Creative Journey:** Session moved from abstract principles to concrete stakeholder needs to process transformation. Each technique built naturally on the previous, creating a coherent product vision rather than a scattered idea list.

## Idea Organization and Prioritization

### Theme 1: Intelligent Dashboard & Alerting
_The system that screams where it hurts_

- **F1** - Automatic detection as foundation, human action as choice
- **BDM1** - Alert-driven dashboard with fires on top
- **BDM2** - Blockage-first partner view: problem, owner, evidence
- **M1** - Deadline-driven backward planning with color codes (green/orange/red)

### Theme 2: Smart Task Architecture
_Tasks that organize themselves_

- **System1** - Task dependencies and automatic handoff
- **System3** - Deliverable chain: tasks produce assets other tasks need
- **E1** - Main tasks with assignable subtasks per responsible person
- **S1** - Conditional tasks (lawyer only on contract deviation)
- **IT1** - Push notifications on task activation

### Theme 3: Partner as Active Participant
_The partner is not an outsider but a collaborator in the process_

- **Partner1** - Own login from contract signing
- **Partner2** - Transparent status view and scheduled dates
- **System4** - Partner uploads feed the system and unblock tasks
- **R1** - Self-service intake flow as front door to the system

### Theme 4: Configuration & Process Optimization
_Smart adaptable without complexity_

- **F4** - Integration choice as path-determining variable
- **S2** - Contract type (Expert/Software) as second variable
- **C1** - Combine Phase 4+5 into Go-to-Market (from 6 to 5 phases)
- **F2** - The phase model as proven backbone

### Theme 5: Context Integration & Information Flow
_All information in the right place without double work_

- **BDM3** - Link external sources (Circleback transcripts, DocuSign)
- **Marketing1** - Tasks with prerequisite assets and visibility of what's missing
- **System2** - Trust-based checkbox completion: lightweight mechanism

### Theme 6: Activity Logging & Personal Task Management (Added post-session)

- **Activity1** - Log activities on any task with assignee and deadline
- **Activity2** - Personal "today" widget (top-right) showing your tasks and deadlines for the day -- Odoo-inspired

### Breakthrough Concepts

1. **Partner self-service intake (R1)** -- transforms contact form into system front door
2. **Deliverable chain with partner uploads (System3 + System4)** -- partner uploads photos -> Marketing auto-unblocked

### Prioritization Results

- **Build Priority:** Themes in order: (1) Dashboard & Alerting, (2) Smart Task Architecture, (3) Partner as Active Participant, (4) Configuration & Process Optimization, (5) Context Integration, (6) Activity Logging & Personal Tasks
- **Quick Wins:** Checkbox completion, standard contract templates, phase combination
- **Most Innovative:** Partner self-service intake, deliverable chain, alert-driven dashboard
